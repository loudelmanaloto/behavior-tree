The Tick Loop

The entire behavior tree is "ticked" (evaluated) once per frame. The signal starts at the Root and travels down.

2

State Propagation

Nodes execute and immediately return a state: SUCCESS, FAILURE, or RUNNING.

3

Handling "Running"

If a node returns RUNNING, the tree pauses that branch. Next frame, it resumes checking that same node until it finishes.

SYSTEM_NOTE

Unlike FSMs which sit in a state, BTs effectively "re-decide" their path every tick, making them highly reactive to changing conditions.
