FSM vs Behavior Trees

ARCHITECTURAL_ANALYSIS: COMPARISON_MATRIX

11

Finite State Machine

THE "OLD SCHOOL" WAY

CRITERIA

Behavior Tree

THE MODERN STANDARD

Flat Graph

States connected by transitions. Cyclic structure.

VS
STRUCTURE

Hierarchical Tree

Directed Acyclic Graph (DAG). Depth-first traversal.

State Explosion

Complexity grows as O(N²). Hard to add new states late.

VS
SCALABILITY

Scales Gracefully

Add subtrees without affecting existing logic. Linear growth.

Tightly Coupled

Logic embedded in transitions. Cannot copy-paste behaviors easily.

VS
REUSE

Modular Subtrees

"Search" subtree can be reused for Guard, Soldier, and Boss.

Explicit Transitions

Must define A → B conditions manually.

VS
LOGIC FLOW

Implicit Priority

Top/Left nodes naturally take priority. No spaghetti wiring.

Simple Concept

Easy to understand: "If this, then that."

VS
LEARNING

Moderate Curve

Requires thinking in trees/tasks. Nodes vs Decorators takes time.
