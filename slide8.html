Execution Flow

Success, Failure,
& Running

SUCCESS return true;

The task completed successfully (e.g., "Enemy Dead", "Path Found"). The parent node advances.

FAILURE return false;

The task failed or is impossible (e.g., "No Ammo", "Path Blocked"). The parent attempts alternatives.

RUNNING return running;

The task is in progress (e.g., "Moving..."). It will be ticked again next frame without blocking code.

State Propagation

These states bubble up the tree, allowing Composite nodes (Sequence/Selector) to control logic flow dynamically.
